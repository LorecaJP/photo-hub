# 撮影＆現像ハブ — 設計書 (v0.1 ドラフト)

> 個人用の「撮る → 現像する → 仕上げる」を一箇所にまとめた静的サイト。
> 既存の Lightroom ガイドを核に、機材ごとの撮り方・使い方・用語リファレンスへ拡張する。

---

## 1. 目的とコンセプト

- **目的**：自分の機材（EOS R8 / DJI Osmo Pocket 4 / DJI Mic Mini）の撮り方・操作を、初心者目線でいつでも引けるリファレンスにする。
- **読者**：自分（写真は趣味、レタッチはこれから）。スマホで見る前提。
- **トーン**：やさしい・断定しすぎない・最初に結論。専門用語は用語集へリンク。
- **コンセプト**：Lightroom の現像ワークスペースを思わせる「ダークな作業台」。1テーマ1ページで、迷わず目的のページに着ける。

---

```

│
├─ 機材で撮る ─────────────────

│   ├─ EOS R8｜操作・使い方         eos-r8/operate.html
│   ├─ Pocket 4｜写真・動画の撮り方  pocket4/shoot.html

│   └─ DJI Mic Mini｜使い方         mic-mini/index.html
│
├─ 現像する ──────────────────
│   └─ Lightroom ガイド            lightroom/index.html  ← 既存ページを移植
│
└─ リファレンス ───────────────
    ├─ 写真・動画 用語集            reference/glossary.html
    └─ 共通ワークフロー             reference/workflow.html
```

### ページの役割分担（重要な切り分け）

| 種類 | 答える問い | 中身の例 |
|---|---|---|
| **撮り方ガイド** | 「どう撮ればいい写真／動画になる?」 | 露出・構図・モード選び・シーン別の設定 |
| **操作・使い方** | 「このボタン／メニューは何?」 | ボタン配置・初期設定・メニュー階層・転送 |
| **リファレンス** | 「この言葉の意味は? 全体の流れは?」 | 用語集・撮影〜書き出しの全体フロー |

> 撮り方ページで用語が出たら用語集へ、操作が必要なら操作ページへ相互リンクする。

## 3. 各ページ詳細仕様

### 3-1. ホーム（ハブ）`index.html`
- ヒーロー：サイトの目的を一言。
- セクション「機材で撮る」：R8 / Pocket 4 / Mic Mini のカード（撮り方・使い方へ2リンク）。
- セクション「現像する」：Lightroom ガイドへの導線。
- セクション「リファレンス」：用語集・ワークフローへの導線。
- 各カードにデバイス別アクセント色（後述）でひと目で区別。

### 3-2. EOS R8｜写真の撮り方 `eos-r8/shoot.html`
- **露出の三角形**：絞り(F) / シャッタースピード(SS) / ISO の関係（図 or インタラクティブ）。
- **撮影モードの選び方**：P / Av(A) / Tv(S) / M / シーン。初心者の入口は「Av優先」。
- **ピント・被写体検出AF**：人物・動物・乗り物検出、瞳AF、フォーカスモード。
- **測光と露出補正**：明るすぎ／暗すぎの直し方。
- **構図の基本**：三分割・日の丸・前ボケ・水平。
- **RAWで撮る設定**：画質設定、RAW+JPEG。
- **シーン別**：パーク日中 / 夜景・イルミ / 料理（マクロ寄り）/ ポートレート・キャラ。
- **インタラクティブ案**：F値・SS・ISO を動かすと「ボケ量・ブレ・ノイズ・明るさ」がプレビューで変わる露出シミュレーター。

### 3-3. EOS R8｜操作・使い方 `eos-r8/operate.html`
- 各部名称（前後ダイヤル・モードダイヤル・マルチコントローラー・シャッター）。
- **最初にやる初期設定**：日付・画質RAW・AF方式・手ブレ補正・オートレビュー等。
- メニュー階層のざっくり地図（撮影／AF／再生／機能設定）。
- ファインダーと液晶（バリアングル）の使い分け。
- カード・バッテリー・充電（USB-C）の扱い。
- **動画の基本操作**（R8は動画も強い）：記録ボタン・解像度/フレームレート選択の入口。
- スマホ転送（Canon Camera Connect）の流れ。

### 3-4. Pocket 4｜写真・動画の撮り方 `pocket4/shoot.html`
- **ジンバルの活かし方**：歩き撮りの基本姿勢、ティルト/パンの見せ方。
- **ActiveTrack 7.0**：被写体追尾でひとり撮り・歩きながらのレポート。
- **基本ショット集**：フォロー / リビール / パン / ローアングル / 自撮り。
- **解像度とフレームレート**：4K30/60の使い分け、4K120/240スロー。
- **D-Log / 色**：そのまま納品系 vs 後で色を作る系。フィルムトーン。
- **タイムラプス / ハイパーラプス / モーションラプス**。
- **音声**：内蔵マイク vs DJI Mic Mini 連携（風切り対策）。
- **シーン別**：パレード・ショー / 食事Vlog / 移動の記録 / 自撮り。
- **インタラクティブ案**：fps とシャッター角度（180°ルール）の関係を示すスロー/質感シミュレーター。

### 3-5. Pocket 4｜操作・使い方 `pocket4/operate.html`
- 各部（画面回転起動・ジョイスティック・ズームボタン・プリセットボタン）。
- タッチ画面の操作（スワイプで設定／撮影モード切替）。
- ジンバルモード（フォロー / FPV / 固定）の切り替え。
- ズーム操作（2xロスレス〜4x）。
- **ストレージ管理**：内蔵107GB の使い方、USB高速転送、Wi-Fi転送。
- DJI Mimo アプリ連携・ファームウェア更新。
- アクセサリー（Mic / フィルライト / 防水ケース）の付け方。

### 3-6. DJI Mic Mini｜使い方 `mic-mini/index.html`（小さめ）
- ペアリング（Pocket 4 / iPhone / カメラ）。
- 装着（クリップ・マグネット）、ウインドジャマー。
- ノイズキャンセル設定、ゲイン。
- 充電ケースとバッテリー運用。

### 3-7. Lightroom ガイド `lightroom/index.html`
- **既存の作り込み済みページをそのまま移植**。ハブのナビ／フッターに合わせて統一。

### 3-8. 写真・動画 用語集 `reference/glossary.html`
- 既存の用語集を独立・拡張。カテゴリ分け（露出 / 色 / 画質 / 動画 / レンズ / ファイル）。
- 各ガイドページの用語からアンカーリンクで飛べるようにする。
- 検索ボックス（クライアントサイドの簡易フィルタ）。

### 3-9. 共通ワークフロー `reference/workflow.html`
- 撮る → 取り込む → 選ぶ → 現像 → 書き出す → 共有 の一本道を図解。
- 機材別の入口（R8はSDカード、Pocket 4はUSB/Wi-Fi）を分岐で表示。

## 4. デザインシステム

既存 Lightroom ガイドのトークンを共通化して全ページで使う。

- **背景** `#191b1f` / パネル `#22252b` / ライン `#363b44`
- **文字** `#eef1f5` / soft `#a7adb8` / faint `#6f7681`
- **基調アクセント（amber）** `#e9a93c`
- **副アクセント（teal）** `#5fc6c0`
- **書体**：見出し Zen Kaku Gothic New / 本文 Noto Sans JP / 数値・ラベル Space Mono

### デバイス別アクセント（区別のため）
| デバイス | アクセント | 意図 |
|---|---|---|
| EOS R8 | amber 系（`#e9a93c`） | 一眼＝写真の主役 |
| Pocket 4 | teal 系（`#5fc6c0`） | 動画・ジンバル |
| Mic Mini | 紫系（`#9b8cff`） | 音声 |
| Lightroom / 共通 | amber 基調 | 現像 |

> ベースは完全共通。アクセントだけ差し替えることで「統一感はあるがページの所属がひと目で分かる」状態にする。

### 共通コンポーネント
- 固定ナビ（ハブ↩・現在地ハイライト）
- ヒーロー / セクション見出し（eyebrow + h2 + lead）
- カード / 折りたたみ（パネル詳解で使用）/ 表 / 用語カード
- インタラクティブ・シミュレーター枠（ページごとに中身を差し替え）

## 5. 技術構成

- **マルチページ静的サイト**（フレームワークなしの素のHTML/CSS/JS）。
- **共通CSS**：`/assets/style.css` を全ページで読み込み（トークン・コンポーネント）。
- **共通JS**：`/assets/main.js`（ナビ・スクロール表示・用語フィルタなど）。
- 各ページ固有のシミュレーターは、そのページ内 `<script>` に閉じる。
- 画像は使わず **CSSで図やサンプルを描く**方針（軽量・著作権フリー・GitHub Pages向き）。
- **ホスティング**：GitHub Pages。**更新**：Claude Code（iPhone）で完結。

### ファイル構成（案）
```
/
├─ index.html
├─ assets/
│   ├─ style.css
│   └─ main.js
├─ eos-r8/
│   ├─ shoot.html
│   └─ operate.html
├─ pocket4/
│   ├─ shoot.html
│   └─ operate.html
├─ mic-mini/
│   └─ index.html
├─ lightroom/
│   └─ index.html
└─ reference/
    ├─ glossary.html
    └─ workflow.html
```

> 注：チャット内プレビューは1ページ単位。共通CSSを使う構成は、実際にGitHub Pagesへ置いて初めて全ページ統一が反映される。プレビュー用に各ページへCSSをインライン同梱する運用も選べる（要相談）。

## 6. 制作フェーズ（おすすめ順）

1. **基盤**：共通CSS/JS＋ホーム（ハブ）を作り、Lightroomガイドを移植して土台を固める。
2. **EOS R8**：撮り方 → 操作の2ページ（露出シミュレーターを実装）。
3. **Pocket 4**：撮り方 → 操作の2ページ（fps/スローのシミュレーター）。
4. **リファレンス**：用語集（検索付き）＋ワークフロー。
5. **Mic Mini** と全体の相互リンク・微調整で仕上げ。

各フェーズごとに1ファイルずつ作って確認 → 次へ、が安全。
